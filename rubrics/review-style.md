# KilaBz Style Review Rubric

1. Naming is clear and consistent with project conventions.
2. Code structure is readable (appropriate decomposition, minimal nesting, and clear control flow).
3. Comments/docs are accurate, necessary, and do not contradict behavior.
4. Formatting and language/framework conventions are followed consistently.
5. Duplication is minimized and abstractions are appropriate for current complexity.
6. Public interfaces and error messages are clear, concise, and maintainable.
